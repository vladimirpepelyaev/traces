# Интеграция раздела «Это обсуждают» — Продуктовый и Технический План

Данный план описывает добавление слоя интеллектуального обнаружения важных обсуждений в сообществе, включая систему усиленных реакций, механизмы ограничения влияния, расчет оценки дискуссий и построение интерактивного интерфейса в рамках существующего стиля платформы.

## 1. Модели данных

### А. Реакции на комментарии (внутри `FeedPost.comments`)
Расширяем структуру каждого комментария в `FeedPost.comments` следующими полями:
- `firesCount?: number` (количество огонёчков)
- `negativeReactions?: number` (количество негативных реакций)
- `positiveAttentionPct?: number` (процент положительного внимания)
- `fireUsers?: { userId: string; userName: string; userAvatar: string; timestamp: string }[]` (список проголосовавших пользователей для контроля уникальности)

### Б. Ограничение влияния пользователя (в `AppUser`)
Каждый пользователь получает свойства (уже добавлены в `AppUser`):
- `boostsLeft?: number` — сколько усилений осталось (дефолт 3)
- `boostsUsed?: number` — сколько потрачено за день
- `boostsResetTime?: string` — время обновления лимита (через 24 часа с момента первой траты)

## 2. Скоринг и формула попадания в «Это обсуждают»

Каждое обсуждение публикации получает интегральный показатель обсуждаемости **Discussion Score**:
```typescript
DiscussionScore = (ScoreActivity * 0.4) + (ScoreQuality * 0.4) + (ScoreBalance * 0.2)
```

### А. Оценка Активности (ScoreActivity)
1. Количество уникальных участников дискуссии (увеличивает вес за счет разнообразия авторов).
2. Общее количество комментариев.
3. Скорость появления комментариев (если есть свежие за последние 48 часов, дается бонус).

### Б. Оценка Качества (ScoreQuality)
1. Количество усиленных реакций `firesCount` во всей ветке комментариев.
2. Процент положительного внимания в комментариях (процент огонёчков `firesCount / (firesCount + negativeReactions) * 100` должен быть выше 85% для попадания в подборку лучших комментов).

### В. Оценка Баланса и Дифференциации точек зрения (ScoreBalance)
Система поощряет наличие конструктивного спора или диалога:
- Если в ветке есть комментарии типов `continue_thought` (продолжение), `disagree` (возражение), и `share_experience` (опыт), то показатель разнообразия мнений максимален.
- Разнообразие мнений рассчитывается от наличия хотя бы одной пары противоположных типов комментов (например, `continue_thought` + `disagree` дает весомый бонус).

### Г. Фильтр свежести
Основной период для анализа — публикации, созданные или получившие активность за последние **14 дней** (анализируется дата публикации или дата последней активности).

## 3. Выбор лучших комментариев внутри темы

Внутри каждого обсуждения алгоритм выявляет до **3 лучших мнений** по категориям:
1. **Лучшее мнение-продолжение**: с наибольшим числом `firesCount` из комментов типа `continue_thought`.
2. **Лучшее мнение-возражение**: с наибольшим числом `firesCount` из комментов типа `disagree`.
3. **Лучшее мнение-опыт**: с наибольшим числом `firesCount` из комментов типа `share_experience`.
4. **Резонансные мнения**: комментарии с самым высоким уровнем вовлеченности (высокий `firesCount`), которые не вписались в дефолтные типы.

## 4. Тематическая группировка (Topics)

Используем существующую систему топиков в проекте (каждая публикация уже содержит `topicScores: { topic: string; score: number }[]`, рассчитанные через `TopicScoreService.analyzePost`).
Группируем публикации по тематикам (например, "Искусственный интеллект", "Экология и общество" и т.д.):
- Выбираем топики с `score > 40%`.
- Агрегируем публикации внутри каждого топика.
- Считаем средний или суммарный рейтинг топика на основе его лучших обсуждений.

## 5. UI раздела «Это обсуждают»

Новый раздел интегрируется в sidebar как `'discussed-now'` («Это обсуждают»).
Интерфейс строится по паттернам VK/Следы:
1. **Главный экран раздела**:
   - Красивая интерактивная плашка с объяснением ценности ("Мы ищем конструктивные аргументы, разные точки зрения и действительно глубокие обсуждения вместо пустого кликбейта").
   - Список горячих тем (карточки топиков). В карточке пишется: её название, сводное объяснение ("Почему тема в фокусе"), количество публикаций внутри неё, общее количество участников дискуссий.
   - Микро-превью лучших комментариев прямо на главной странице топиков для быстрого погружения в суть спора.
2. **Экран конкретной темы (детальный вид)**:
   - Список публикаций по теме, получивших наибольший `DiscussionScore`.
   - Специальное "Продуктовое кольцо мнений" (визуальный разрез из трех колонок или вкладок): "Глубокое продолжение" (Teal), "Альтернативный взгляд / Оппоненты" (Red), "Реальный опыт" (SeaBlue).
   - Интерактивный UI реакций прямо на этой странице для вовлечения пользователей в оценку качественного контента.
